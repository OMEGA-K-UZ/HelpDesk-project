# 📘 Установка HelpDesk v2 — полная инструкция для новичка

Эта инструкция проведёт тебя от чистого сервера до работающей системы.
Делай шаг за шагом, ничего не пропускай. Команды копируй целиком.

---

## ⚠️ Что тебе понадобится

1. **Сервер** с Ubuntu 22.04 или 24.04 (подойдёт VPS, виртуалка или физический).
2. **Доступ к серверу** по SSH (логин и пароль или ключ).
3. **Архив** `helpdesk-v2-release.tar.gz` (твой дистрибутив).
4. Примерно **20 минут** времени.

> 💡 Все команды вводятся в терминале сервера. После каждой команды жми Enter.
> Если команда просит пароль — вводи пароль своего пользователя (символы не отображаются — это нормально).

---

## ШАГ 1. Подключись к серверу

На своём компьютере открой терминал и подключись (замени `ВАШ_IP` на адрес сервера):

```bash
ssh ops@ВАШ_IP
```

Если просит подтверждение — напиши `yes` и Enter. Введи пароль.

> 💡 `ops` — это имя пользователя. Если у тебя другое — подставь своё.

---

## ШАГ 2. Установи нужные программы

HelpDesk работает на **Node.js** (движок) и хранит данные в **PostgreSQL** (база данных).
Установим обе программы.

### 2.1. Обнови список пакетов
```bash
sudo apt update
```

### 2.2. Установи PostgreSQL (база данных)
```bash
sudo apt install -y postgresql postgresql-contrib
```

Проверь что установилось:
```bash
psql --version
```
Должна появиться строка вроде `psql (PostgreSQL) 16.x`.

### 2.3. Установи Node.js 20 (движок)
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

Проверь:
```bash
node -v
```
Должно показать `v20.x.x` (или выше).

### 2.4. Установи PM2 (чтобы сервер работал постоянно)
```bash
sudo npm install -g pm2
```

---

## ШАГ 3. Создай базу данных

База данных — это «хранилище» всех тикетов, пользователей, настроек.
Создадим пустую базу и пользователя для неё.

Скопируй ВЕСЬ блок целиком и вставь в терминал (он выполнится как одна команда).
**Замени `МОЙ_ПАРОЛЬ123` на свой пароль** (придумай надёжный, запиши его — он понадобится дальше):

```bash
sudo -u postgres psql << SQL
CREATE DATABASE helpdesk;
CREATE USER helpdesk_user WITH PASSWORD 'МОЙ_ПАРОЛЬ123';
GRANT ALL PRIVILEGES ON DATABASE helpdesk TO helpdesk_user;
ALTER DATABASE helpdesk OWNER TO helpdesk_user;
SQL
```

Если всё прошло — увидишь несколько строк `CREATE` / `GRANT` / `ALTER`. Ошибок быть не должно.

> 💡 Запомни три вещи (они уже заданы выше):
> - Имя базы: `helpdesk`
> - Пользователь базы: `helpdesk_user`
> - Пароль: тот, что ты вписал вместо `МОЙ_ПАРОЛЬ123`

---

## ШАГ 4. Загрузи проект на сервер

### Вариант А — архив уже на сервере
Если ты уже скопировал `helpdesk-v2-release.tar.gz` на сервер (например через SFTP) — пропусти к Шагу 5.

### Вариант Б — загрузить с компьютера
На СВОЁМ компьютере (не на сервере!) открой новый терминал и выполни
(замени путь к файлу и IP):

```bash
scp ~/Downloads/helpdesk-v2-release.tar.gz ops@ВАШ_IP:~/
```

Это скопирует архив в домашнюю папку сервера.

---

## ШАГ 5. Распакуй проект

Вернись в терминал сервера. Распакуй архив:

```bash
cd ~
mkdir -p helpdesk-v2
tar -xzf helpdesk-v2-release.tar.gz -C helpdesk-v2
cd helpdesk-v2
```

Проверь что файлы на месте:
```bash
ls
```
Должны быть: `src`, `public`, `package.json`, `install.sh`, миграции `migrate-*.js` и другие.

---

## ШАГ 6. Запусти установщик (первый раз)

```bash
./install.sh
```

При первом запуске он создаст файл настроек `.env` из шаблона и остановится,
попросив его отредактировать. Это нормально.

---

## ШАГ 7. Впиши свои настройки

Открой файл настроек в редакторе:

```bash
nano .env
```

Ты увидишь несколько строк. Отредактируй ТРИ из них:

1. **`DB_PASS=ЗАМЕНИ_НА_ПАРОЛЬ_БД`**
   → впиши пароль базы из Шага 3 (вместо `ЗАМЕНИ_НА_ПАРОЛЬ_БД`).
   Получится например: `DB_PASS=МОЙ_ПАРОЛЬ123`

2. **`APP_URL=https://office.example.uz`**
   → впиши адрес, по которому будет открываться система.
   Если домена пока нет — впиши `http://ВАШ_IP:4000`

3. **`SESSION_SECRET=ЗАМЕНИ_НА_СЛУЧАЙНУЮ_СТРОКУ`**
   → впиши любой длинный набор букв и цифр (например `kf83jdLs92mfksal28dhsknxm`).
   Это «соль» для защиты сессий, никому её не показывай.

**Как сохранить в nano:**
- Нажми `Ctrl + O`, затем `Enter` (сохранить)
- Нажми `Ctrl + X` (выйти)

## ШАГ 8. Запусти установщик (второй раз)

Теперь установщик поставит зависимости и создаст все таблицы в базе:

```bash
./install.sh
```

Ты увидишь как:
- Устанавливаются npm-пакеты (может занять 1-2 минуты)
- Создаётся папка для файлов
- Применяются миграции (создание таблиц) — много строк ✅

В конце появится:
```
✅ Установка завершена!
```

## ШАГ 9. Запусти систему

```bash
pm2 start src/server.js --name helpdesk
```

Проверь что работает:
```bash
pm2 logs helpdesk
```

Должны появиться строки:
```
🚀 eMaktab HelpDesk v2 → http://localhost:4000
🔑 admin@helpdesk.local / admin123
```

Нажми `Ctrl + C` чтобы выйти из просмотра логов (сама система продолжит работать).

### Сделай автозапуск (чтобы система включалась после перезагрузки сервера):
```bash
pm2 save
pm2 startup
```

Последняя команда выведет строку, начинающуюся с `sudo env PATH=...`.
**Скопируй её целиком и выполни** — это включит автостарт.

## ШАГ 10. Зайди в систему

Открой браузер на своём компьютере и перейди по адресу:

- Если есть домен: `https://твой-домен`
- Если только IP: `http://ВАШ_IP:4000`

**Данные для входа:**
- Email: `admin@helpdesk.local`
- Пароль: `admin123`

> 🔒 **ВАЖНО:** сразу после входа смени пароль!
> Профиль (внизу слева) → Сменить пароль.

# 🌐 (Необязательно) Домен и HTTPS через Nginx

Если хочешь открывать систему по красивому адресу `https://office.твой-домен`
вместо `http://IP:4000`, настрой Nginx.

### Установи Nginx:
```bash
sudo apt install -y nginx
```

### Создай конфиг:
```bash
sudo nano /etc/nginx/sites-available/helpdesk
```

Вставь (замени `office.твой-домен` на свой адрес):
```nginx
server {
    listen 80;
    server_name office.твой-домен;

    client_max_body_size 25M;

    location / {
        proxy_pass http://localhost:4000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```
Сохрани (`Ctrl+O`, `Enter`, `Ctrl+X`).

### Включи конфиг:
```bash
sudo ln -s /etc/nginx/sites-available/helpdesk /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### Добавь бесплатный HTTPS-сертификат:
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d office.твой-домен
```
Следуй подсказкам (введи email, согласись с условиями).
Сертификат обновляется автоматически.

> ⚠️ Для домена нужно чтобы DNS-запись `office.твой-домен` указывала на IP сервера.

# ❓ Если что-то пошло не так

### «Error: listen EADDRINUSE :::4000» (порт занят)
Старый процесс висит на порту. Освободи и запусти заново:
```bash
sudo fuser -k 4000/tcp
pm2 restart helpdesk
```

### «password authentication failed» (не подключается к базе)
Пароль в `.env` (DB_PASS) не совпадает с тем, что задал в Шаге 3.
Открой `nano .env`, исправь пароль, потом `pm2 restart helpdesk`.

### Сайт не открывается по IP:4000
Возможно фаервол блокирует порт. Открой его:
```bash
sudo ufw allow 4000
```

### «relation ... does not exist» (нет таблицы)
Миграции не применились. Запусти их вручную:
```bash
cd ~/helpdesk-v2
node src/migrate.js
for m in migrate-*.js; do node "$m"; done
pm2 restart helpdesk
```

### Забыл пароль администратора
Сбрось на `admin123`:
```bash
cd ~/helpdesk-v2
node -e "console.log(require('bcryptjs').hashSync('admin123',10))"
```
Скопируй полученную строку (начинается с `\$2a\$10\$`) и вставь вместо ХЕШ:
```bash
sudo -u postgres psql -d helpdesk -c "UPDATE users SET password_hash='ХЕШ' WHERE role='admin';"
```
Теперь вход: твой админский email / `admin123`.

# 📦 Как перенести систему на ДРУГОЙ сервер потом

На старом сервере собери свежий архив:
```bash
cd ~/helpdesk-v2
bash build-release.sh
```
Получишь `~/helpdesk-v2-release.tar.gz`. Перенеси его на новый сервер
и повтори эту инструкцию с Шага 1.

> 💡 Чтобы перенести и ДАННЫЕ (тикеты, настройки), а не только программу,
> на старом сервере сделай дамп базы:
> ```bash
> pg_dump -U helpdesk_user helpdesk > ~/helpdesk-backup.sql
> ```
> А на новом, ПОСЛЕ установки, восстанови:
> ```bash
> psql -U helpdesk_user -d helpdesk < ~/helpdesk-backup.sql
> ```
